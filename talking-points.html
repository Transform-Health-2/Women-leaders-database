Open challenges

🔗 LinkedIn URL scraping — why it won't work

The current Manage Profile flow uses a LinkedIn URL as an identity token to look up who is requesting a profile change. The intent was to also auto-fill profile data by scraping the LinkedIn page — saving leaders from re-entering their role, organisation, and photo.

The hard reality: LinkedIn aggressively blocks all automated scraping. Their terms of service prohibit it, they serve login walls to headless browsers, they rotate bot-detection fingerprinting, and they have taken legal action against scrapers (hiQ v. LinkedIn, 2022). Any scraper breaks within days of being built and puts the project at legal risk.

Practical alternatives: (1) Ask users to paste their LinkedIn URL as a vanity identifier only — no scraping, just a hyperlink. (2) Use the LinkedIn API (requires partner approval, significant overhead). (3) Let users fill the form themselves — which is what the current Submit flow already does well.

⚠️ Firebase Storage for photo uploads — proceed with caution

Firebase Storage is already wired up and working for photo uploads in the Submit flow. It's convenient, but there are real risks to understand before going to production.

Cost exposure
Firebase Storage bills per GB stored and per GB downloaded. If the directory grows to hundreds of profiles and gets traffic, costs can spike unexpectedly — especially with no CDN caching layer in front.

No moderation gate
Anyone who submits the form can upload any image. Without an admin review step, inappropriate or oversized files go straight into the bucket and are publicly readable.

Security rules
The default Firebase Storage rules are often left open during dev. Before launch, rules must restrict writes to authenticated users only and enforce file-type and size limits.

Recommendation
Keep Firebase for now but: (1) add file-type + 2 MB size validation client-side (compression util exists), (2) add admin approval before a photo URL is written to the profile record, (3) set a monthly budget alert in the Firebase console.
