# Data Quality Standards — Transform Health Women Leaders Directory

This document defines what makes a **high-quality, publishable profile** in the Transform Health Directory. It's used by:
- **Admin reviewers** — to approve/reject submissions consistently
- **Content contributors** — to understand expectations before submitting
- **Quality assurance team** — to maintain directory standards

---

## Overview

A profile moves from **PENDING** → **LIVE** only when it meets these standards:

| Standard | Why | Guideline |
|----------|-----|-----------|
| **Completeness** | Readers get a full picture | All required fields filled |
| **Accuracy** | Directory is trustworthy | Information verified or verifiable |
| **Relevance** | Directory stays focused | Leader's work aligns with digital health |
| **Appropriateness** | Directory is professional | Content follows community standards |
| **Clarity** | Profiles are useful | Writing is clear and professional |

---

## Required Fields — Must Be Present

These fields **must** be filled for a profile to be approved:

| Field | Status | Standard |
|-------|--------|----------|
| First Name | **REQUIRED** | Non-empty, matches a person's name |
| Last Name | **REQUIRED** | Non-empty, matches a person's name |
| Email | **REQUIRED** | Valid email address (for admin contact only) |
| Current Role | **REQUIRED** | Specific job title (not vague like "Manager") |
| Organization | **REQUIRED** | Full, recognizable organization name |
| Expertise | **REQUIRED** | At least 2-3 areas selected; must be from approved list or custom entry |
| Bio | **REQUIRED** | Minimum 2-3 sentences; maximum 500 characters |
| Primary Country | **REQUIRED** | Valid country name |
| Geographical Scope | **REQUIRED** | Local, National, Regional, or Global |

## Field-by-Field Standards

### Name (First + Last)
**Standard:**
- ✓ Matches person's real name (as they introduce themselves professionally)
- ✓ No titles (Dr., Prof., Hon., Mr., Mrs.)
- ✓ No all-caps unless that's how the person writes it
- ✓ Proper spelling (verifiable against LinkedIn or official bios)

**Examples:**
- ✓ Amara Okonkwo
- ✓ Dr. Maria Chen (→ should be "Maria Chen")
- ✓ Jean-Luc Dubois

**Reject if:**
- ✗ Only first name or last name provided
- ✗ Obvious typos (e.g., "Jonh" for "John")
- ✗ Name includes degrees, titles, or honorifics
- ✗ Name is a company, organization, or brand

---

### Email Address
**Standard:**
- ✓ Valid email format (person@domain.com)
- ✓ Should match organization (optional but preferred)
- ✓ Not a temporary/disposable email
- ✓ Ideally professional domain (not temp email)

**Examples:**
- ✓ amara.okonkwo@who.int
- ✓ maria@healthinnovations.org
- ✓ jean.dubois@gmail.com (acceptable if that's their primary email)

**Accept but flag if:**
- ⚠ Gmail/Yahoo but no alternate — assume it's verified when user consents
- ⚠ Generic domain (info@, contact@) — might bounce

**Reject if:**
- ✗ Invalid format (no @, or no domain)
- ✗ Obviously fake (e.g., test@test.test)
- ✗ Temporary email service (tempmail.com, etc.)

---

### Current Role / Job Title
**Standard:**
- ✓ Specific, professional job title
- ✓ Reflects current position or primary role
- ✓ 3-50 characters
- ✓ Not promotional or vague

**Examples:**
- ✓ "AI Research Lead"
- ✓ "Head of Digital Health Transformation"
- ✓ "Health Policy Director"
- ✓ "Founding Director, Telemedicine Initiative"
- ✓ "Independent Digital Health Consultant"

**Reject if:**
- ✗ Too vague: "Manager," "Staff," "Leader," "Employee"
- ✗ Not a real title: "Digital Health Enthusiast," "Innovator"
- ✗ Too long/promotional: "Chief Disruptor of Healthcare & AI Transformation Guru"
- ✗ Outdated (e.g., role from 10+ years ago)

---

### Organization
**Standard:**
- ✓ Full, official organization name (not abbreviation)
- ✓ Verifiable (check LinkedIn, Google, or org website)
- ✓ 2-100 characters
- ✓ Matches person's current affiliation if possible

**Examples:**
- ✓ "World Health Organization (WHO)"
- ✓ "Gates Foundation"
- ✓ "Ministry of Health, Kenya"
- ✓ "MIT D-Lab"
- ✓ "Independent Consultant"

**Accept if:**
- ⚠ University listed generically ("Harvard University" instead of "Harvard School of Public Health") — acceptable

**Reject if:**
- ✗ Too vague: "Company," "Organization," "Tech"
- ✗ Unverifiable: "Secret Government Agency" or obvious fake
- ✗ Clearly outdated: Last verified more than 2 years ago and person's LinkedIn shows different role

---

### Expertise Tags
**Standard:**
- ✓ 2-5 tags selected
- ✓ At least 80% of tags must be from the approved list
- ✓ Custom tags (if used) are specific and relevant to digital health
- ✓ Tags logically relate to the person's role and bio

**Approved Expertise Areas:**
- AI / Machine Learning
- Digital health
- Health financing
- Health information systems
- Health systems strengthening
- mHealth
- Digital health policy
- Digital health strategy
- Digital health advocacy
- Digital health innovation
- Digital health transformation
- Digital health philanthropy
- Research
- Telemedicine
- Health workforce

**Custom Tag Guidelines:**
- If person selects "Other," they provide a custom tag (max 30 characters)
- Custom tags should be specific: "Maternal Health AI" or "Policy for Digital Equity"
- Vague custom tags are rejected: "Stuff," "Things," "Cool things"

**Examples:**
- ✓ Tags: [AI, Digital Health, Research]
- ✓ Tags: [Digital Health Policy, Health Systems Strengthening]
- ✓ Tags: [Telemedicine, Health Workforce] + custom "Rural Health Deployment"

**Reject if:**
- ✗ Zero tags selected
- ✗ More than 5 tags
- ✗ Tags are unrelated (e.g., [Finance, Pizza, Gardening])
- ✗ Tags don't align with bio or role
- ✗ Custom tag is vague or non-professional

---

### Years of Experience (Optional)
**Standard:**
- ✓ Reasonable number (1-60 years)
- ✓ Roughly aligns with role and organization history
- ✓ Can be a range ("15+" or "20-25 years")

**Examples:**
- ✓ 8
- ✓ 15+
- ✓ 20-25

**Flag if:**
- ⚠ Entry-level role claims 30+ years (doesn't match role)
- ⚠ Senior title claims 2 years (possible but unusual)

**Reject if:**
- ✗ Non-numeric (e.g., "Many years")
- ✗ Unreasonable (500, 0, negative numbers)

### Bio (Biography / Professional Summary)
**Standard:**
- ✓ 2-5 sentences (aim for 150-300 characters)
- ✓ First person ("I") or third person, but consistent
- ✓ Specific and evidence-based
- ✓ Shows impact or contribution
- ✓ No promotional language or hype
- ✓ Professional tone
- ✓ Clear connection to digital health

**GOOD Bio Examples:**
- ✓ "I lead the AI research team at XYZ Foundation, where we develop machine learning models to improve maternal health diagnosis in low-resource settings. I've published 8 peer-reviewed papers and trained 50+ researchers across East Africa. My focus is making AI tools accessible and equitable."

- ✓ "As Health Policy Director for the Ministry of Health, I design and implement digital health policies for 15 African countries. I've led 3 successful health IT implementations and currently chair the Regional Digital Health Committee."

- ✓ "Founder of TeleMed Solutions, connecting rural patients with doctors via mobile telemedicine. Since 2020, we've served 100,000+ patients across 8 countries."

**RED FLAGS — Check Carefully:**
- ⚠ "Passionate about digital health" (everyone says this — needs specifics)
- ⚠ "Expert in everything" (too broad, not credible)
- ⚠ Uses marketing language ("Disrupting," "Revolutionary," "Game-changing")
- ⚠ Claims not verifiable ("Helped millions," "World-renowned")
- ⚠ Too much self-promotion ("Award-winning," "Best in class")

**BAD Bio Examples:**
- ✗ "I love digital health."
- ✗ "I'm experienced in many areas of health."
- ✗ "I'm a digital health revolutionary disrupting the sector."
- ✗ "I'm the world's leading expert in everything related to digital health." (not credible)
- ✗ Completely off-topic: "I'm a digital marketer interested in health products."

**What to Do If Bio Is Weak:**
1. **Contact submitter** — Send a note asking for more specific, impact-focused details
2. **Give an example** — Suggest structure: role + impact + scope
3. **Allow resubmission** — Let them edit and resubmit
4. **Reject if they don't respond** — After 1 week, auto-reject

### Photo
**Standard (Optional but Recommended):**
- ✓ Professional headshot
- ✓ Clear, well-lit, in focus
- ✓ Person's face clearly visible
- ✓ Recent (within 2-3 years)
- ✓ JPG or PNG, less than 5MB
- ✓ 300x300px minimum (800x800 preferred)

**GOOD Photo Examples:**
- ✓ Professional headshot (passport-style)
- ✓ Office or professional setting
- ✓ Smiling, approachable
- ✓ Business casual or professional dress

**RED FLAGS:**
- ⚠ Very old photo (recognizable from decade-old photos but looks very different)
- ⚠ Heavy filters / heavily edited
- ⚠ Informal (selfie, vacation photo)

**BAD Photo Examples:**
- ✗ Group photo
- ✗ Logo or brand image (not a person)
- ✗ Photo of someone else's face
- ✗ No face visible
- ✗ Very low resolution (pixelated)
- ✗ Inappropriate content

**What to Do If Photo Is Missing or Bad:**
- ✓ **Missing:** Approve anyway (photo is optional)
- ✗ **Wrong file type:** Request JPG/PNG
- ✗ **Not a headshot:** Ask for professional photo
- ✗ **Multiple people:** Request headshot instead

### Primary Country
**Standard:**
- ✓ Valid country name (use dropdown)
- ✓ Where person is based or primary work location
- ✓ Should be reasonably current (not from 10+ years ago)

**Examples:**
- ✓ Kenya (person is based here, does regional work)
- ✓ USA (person is based in Seattle, works globally)
- ✓ India (person based in New Delhi, primary work location)

**Reject if:**
- ✗ Invalid country name
- ✗ Country doesn't exist or misspelled
- ✗ Significantly outdated (person moved countries, not updated)

---

### Geographical Scope
**Standard:**
- ✓ One of: Local, National, Regional, Global
- ✓ Matches the scale of their work

**Guidelines:**
- **Local** — Work limited to one city or small region
- **National** — Work in one country
- **Regional** — Work across multiple countries (2-5) or a continent
- **Global** — Work spans multiple continents

**Examples:**
- ✓ Role: "Telemedicine Lead in rural Kenya" → Scope: **Local**
- ✓ Role: "National Health IT Director" → Scope: **National**
- ✓ Role: "Regional AI Lead for East Africa" → Scope: **Regional**
- ✓ Role: "Global Health Officer at WHO" → Scope: **Global**

**What to Do If Scope Doesn't Match:**
- Flag mismatch and ask submitter to clarify
- Examples: "You listed Global scope but bio mentions only Kenya" → Ask to clarify

---

### Countries of Operation (Optional)
**Standard:**
- ✓ If filled, list 1-15 countries
- ✓ Should align with geographical scope and bio
- ✓ No duplicates

**Examples:**
- ✓ Regional scope + countries: Kenya, Uganda, Tanzania, Rwanda, Ethiopia
- ✓ Global scope + countries: USA, UK, Kenya, India, Brazil, Australia

**What to Do If Populated:**
- Spot-check against bio — does it match?
- If "Global" scope but lists only 2 countries, ask to clarify

---

### LinkedIn / Website Links (Optional)
**Standard:**
- ✓ If provided, must be active and valid URL
- ✓ Should point to person's actual profile or website
- ✓ Helps verify profile authenticity

**Examples:**
- ✓ https://www.linkedin.com/in/amaraokonkwo
- ✓ https://www.mywebsite.com/about

**What to Do:**
- ✓ **Click links** — Verify they work and match the profile
- ✗ **Broken link:** Ask for corrected URL
- ✗ **Wrong person's profile:** Reject or ask for clarification
- ✓ **No link:** Acceptable, not required

## Digital Health Relevance — Critical Gate

Before approving ANY profile, verify the person qualifies as a "digital health leader":

### ✓ Clearly Qualifies:
- Works on health IT, digital tools, telemedicine, AI/ML in health
- Holds leadership role in organization focused on digital transformation
- Conducts research on health data or digital systems
- Leads policy or strategy around health informatics
- Manages teams building health technology

### ⚠ Borderline — Needs Clarification:
- Works in healthcare (doctor, nurse) but no explicit digital health role
- Works in health financing but no tech/digital component mentioned
- Works in tech but applies it to health problems (need to verify)
- **Action:** Ask for clarification in bio — how does their work connect to "digital" health?

### ✗ Clearly Doesn't Qualify:
- Works in pharmaceutical sales (not digital health)
- Works in traditional healthcare only (no tech/digital component)
- Works in adjacent fields (e.g., agriculture, education — unrelated to health)
- No clear connection to health OR digital transformation
- **Action:** Reject with explanation

## What Gets Rejected?

| Reason | Example | Action |
|--------|---------|--------|
| **Incomplete** | Missing bio, no expertise selected | Request completion, allow resubmission |
| **Not Digital Health** | Person is a cardiologist with no tech background | Reject, suggest they add details on any health IT work |
| **Spam/Fake** | Nonsensical profile, unverifiable info | Reject immediately |
| **Duplicate** | Person already in directory under same/similar name | Contact submitter — merge or clarify |
| **Inappropriate Content** | Bio contains offensive language or adult content | Reject immediately |
| **Unverifiable** | Claims major role but no LinkedIn/online presence | Friendly rejection — ask them to add LinkedIn link |

## Admin Review Checklist

Use this before clicking **Approve**:

### ✓ Pre-Approval Checklist

- [ ] **Completeness** — All required fields filled?
- [ ] **Accuracy** — Information sounds accurate? (spot-check LinkedIn if linked)
- [ ] **Relevance** — Person qualifies as digital health leader?
- [ ] **Clarity** — Bio is clear and professional?
- [ ] **Appropriateness** — No offensive, promotional, or spam content?
- [ ] **Duplicates** — Is this a duplicate of existing profile? (check similar names)
- [ ] **Links Work** — Do LinkedIn/website links work and match the person?
- [ ] **Photo OK** — If photo present, is it professional? (or acceptable to approve without)

**If YES to all above → APPROVE**  
**If NO to any → Request edits or REJECT with clear reason**

## Continuous Monitoring

After approval, profiles are monitored for:

| Issue | Action |
|-------|--------|
| Spam reports (multiple users report as inappropriate) | Flag for review, possible removal |
| Broken links (LinkedIn/website no longer works) | Flag, send email to leader requesting update |
| Duplicate discovered (same person submitted under different name) | Merge profiles, keep one active |
| Leader requests deletion | Remove immediately |
| Profile outdated (role/org very old, no recent verification) | Not removed, but flagged in internal notes |
